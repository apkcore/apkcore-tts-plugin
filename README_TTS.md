# Obsidian TTS 插件使用说明

## ✅ 功能已实现

本插件支持两种 TTS 引擎：

### 1. **浏览器原生 TTS**（默认，推荐）
- ✅ **稳定可靠** - 直接使用浏览器内置的 Web Speech API
- ✅ **无需网络** - 完全本地运行
- ✅ **兼容性好** - 在 Obsidian 中完美运行
- ⚠️ **音质一般** - 取决于操作系统自带的 TTS 引擎
- ⚠️ **语音选项有限** - 取决于系统可用的语音包

### 2. **EdgeTTS**（实验性）
- ✅ **音质优秀** - 微软 Edge 浏览器的云端 TTS
- ✅ **语音丰富** - 多种中文语音可选
- ❌ **连接问题** - 在 Obsidian 中 WebSocket 连接被阻止
- ⚠️ **需要网络** - 需要连接到 speech.platform.bing.com

## 🚀 使用方法

### 基础朗读
1. 在编辑器中选中要朗读的文本
2. 打开命令面板（`Ctrl+P` 或 `Cmd+P`）
3. 执行 **"朗读选中文本"** 命令
4. 听到语音朗读

### 播放控制
- **停止朗读** - 立即停止当前播放
- **暂停/继续朗读** - 暂停或继续播放
- **切换 TTS 引擎** - 在浏览器TTS和EdgeTTS之间切换

### 设置
打开 设置 → 插件选项 → 本插件：
- **语音** - 选择 EdgeTTS 语音（仅 EdgeTTS 引擎有效）
- **语速** - 调整朗读速度（-50% 到 +100%）
- **最大文本长度** - 限制单次朗读的字符数

## 🔧 引擎切换

插件**默认使用浏览器TTS**，因为它在 Obsidian 中最稳定。

如果要尝试 EdgeTTS（更好的音质）：
1. 执行命令 **"切换 TTS 引擎"**
2. 切换到 EdgeTTS
3. 如果连接失败，会提示你切换回浏览器TTS

## ⚠️ EdgeTTS 连接问题

由于 Obsidian 的 Electron 安全策略，WebSocket 连接到外部服务器可能被阻止。
目前观察到的错误：
- `WebSocket closed: 1006` - 连接被拒绝

**可能的原因：**
1. Obsidian 的渲染进程不允许直接建立外部 WebSocket 连接
2. Electron 的内容安全策略(CSP)限制
3. 缺少必要的 HTTP 请求头（原生 WebSocket API 不支持自定义请求头）

**解决方案：**
- ✅ 使用浏览器原生 TTS（已实现，默认启用）
- 🔄 等待 Obsidian 官方支持或社区找到解决方案
- 🔄 使用 Obsidian Mobile API（移动端可能有不同的限制）

## 🎯 推荐配置

### Windows 用户
- 使用浏览器TTS
- 安装微软语音包以获得更好的中文支持
- 调整语速到 +10% ~ +20% 获得更自然的节奏

### macOS 用户
- 使用浏览器TTS
- macOS 自带的中文语音质量较好
- 可以在系统偏好设置中安装更多语音

## 🐛 调试

如果遇到问题：
1. 打开开发者工具（`Ctrl+Shift+I` 或 `Cmd+Option+I`）
2. 切换到 Console 标签
3. 执行命令并查看错误信息
4. 尝试执行 **"测试 EdgeTTS 连接"** 查看详细的连接日志
